# دليل المساهمة في awesome-mortakaz

مستودع **awesome-mortakaz** هو قائمة منسقة للمشاريع التقنية المنشورة على منصة مرتكز والمفتوحة المصدر.

## شروط قبول المشاريع

لكي يُقبل أي مشروع داخل المستودع يجب أن يستوفي جميع الشروط التالية:

* أن يكون المشروع منشورًا على منصة Mortakaz.
* أن يكون المشروع مفتوح المصدر.
* أن يحتوي على مستودع مصدر عام (GitHub أو GitLab أو ما شابه).
* أن يتضمن رابط صفحة المشروع على Mortakaz.
* أن يحتوي على وصف مختصر وواضح للمشروع.

المشاريع التي لا تحقق هذه الشروط لن يتم قبولها.

---

## بنية المستودع

يتم تنظيم المشاريع حسب أصحابها.

مثال:

```md
# اسم المساهم

- [اسم المشروع](رابط-مرتكز): وصف مختصر للمشروع. - [رابط المستودع](رابط-المستودع)
```

مثال فعلي:

```md
# زكرياء العلوي

- [بيئة زيكو جي إس](https://www.mortakaz.com/projects/6985c7ae76d5a85c6107f577): نظام متكامل بلغة جافاسكريبت لتطوير تطبيقات الويب. - [رابط المستودع](https://github.com/zikojs/ziko)
```

---

## كيفية إضافة مشروع

1. قم بعمل Fork للمستودع.
2. أضف اسمك إلى الفهرس الرئيسي إذا لم يكن موجودًا.
3. أضف المشروع داخل قسمك أو أنشئ قسمًا جديدًا باسمك.
4. تأكد من وجود:

   * رابط Mortakaz.
   * رابط المستودع.
   * وصف مختصر.
5. أرسل Pull Request.

---

## قالب إضافة مشروع

انسخ القالب التالي:

```md
# اسم المساهم

- [اسم المشروع](رابط-مرتكز): وصف مختصر للمشروع. - [رابط المستودع](رابط-المستودع)
```

---

## المساهمات المقبولة

* إضافة مشاريع جديدة تستوفي الشروط.
* تحديث روابط Mortakaz أو المستودعات.
* تحسين أو تصحيح أوصاف المشاريع.
* تحسين تنظيم المستودع.

---

## المساهمات غير المقبولة

لا تقم بإضافة:

* مشاريع غير منشورة على Mortakaz.
* مشاريع مغلقة المصدر.
* مشاريع بدون مستودع مصدر عام.
* مشاريع بدون صفحة Mortakaz.
* مشاريع مكررة.
* روابط دعائية أو تسويقية لا تمثل مشروعًا حقيقيًا.

---

## مراجعة المساهمات

قد يتم رفض Pull Request إذا:

* لم تتوفر روابط Mortakaz والمستودع معًا.
* كان المشروع مغلق المصدر.
* كان الوصف غير واضح أو ناقصًا.
* خالف المشروع هدف المستودع أو هيكله التنظيمي.

شكراً لجميع المساهمين في دعم المشاريع المفتوحة المصدر على Mortakaz.
